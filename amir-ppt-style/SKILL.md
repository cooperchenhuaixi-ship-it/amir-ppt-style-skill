# Amir PPT Style

## Purpose

Use this skill to design presentations with an Amir-inspired visual language: oversized editorial typography, strong page-by-page color rhythm, premium tech gradients, compact narrative pacing, and sparse but sharp commercial storytelling.

This skill is intentionally portable. It does not depend on a specific computer path, source PPT file, image asset, font package, or presentation tool. Any agent that can read skills and create or edit slides can use it.

## Compatibility Contract

If the agent supports skill folders:

1. Load this `SKILL.md`.
2. Read `references/style-system.md` before choosing layouts, colors, or typography.
3. Use the active slide-generation tool available in the environment.

If the agent cannot load reference files:

1. Use the "Quick Style Spec" below as the complete style guide.
2. Treat `references/style-system.md` as optional extended detail, not a required dependency.

If the agent cannot generate PPTX files directly:

1. Produce a slide-by-slide design plan, HTML deck, image storyboard, or PowerPoint implementation instructions using this style.
2. Preserve the same narrative structure, typography scale, and visual routes.

## Workflow

1. Understand the deck job: pitch, proposal, launch, investor, media kit, campaign, portfolio, or strategy.
2. Choose one primary route:
   - `dark-tech`: deep black, blue/violet neon, glassy data blocks, product/investor/campaign energy.
   - `bold-proposal`: one vivid flat background per slide, oversized editorial titles, proposal/campaign energy.
   - `soft-media`: white or black editorial canvas, pastel glow gradients, creator/media kit energy.
3. Build a short, paced deck rather than a dense report. Prefer 6-10 slides unless the user requests otherwise.
4. Give each slide one job: cover, objective, insight, market, mechanism, proof, timeline, economics, ask, or closing line.
5. Make the main headline or metric the dominant visual object on most slides.
6. Verify the final slides visually: no accidental wrapping, no crowded cards, no weak final slide.

## Quick Style Spec

### Core DNA

- Use giant title typography as the main visual object: usually 72-180pt on a 16:9 slide.
- Pair huge titles with tiny section labels: 14-18pt, top-left, often numbered like `04 · CHANNEL MIX`.
- Use modern geometric fonts when available: Geist, Inter, DM Sans, Space Grotesk, Fraunces, or close equivalents.
- Let one visual motif carry the deck: neon ribbons, dot matrices, blurred glow, large cropped typography, or bold flat color.
- Keep body copy short. If a slide needs paragraphs, split the thought across slides.
- Use cards as supporting instruments only: translucent, quiet, and secondary to the headline or metric.
- Close with a large direct line, not a generic "Thank you" slide.

### Route: dark-tech

Use for AI, SaaS, automation, fintech, investor decks, product launches, and serious strategy work.

- Base colors: `#0A0A0A`, `#14101F`, `#0B0B0B`
- Accent colors: `#4D5BFF`, `#549EFF`, `#4FA8FF`
- Supporting text: `#8FA3C8`, `#FFFFFF`
- Visuals: neon ribbons, blue/violet glow, dot matrices, translucent panels, metric blocks
- Feel: premium, intelligent, cinematic, technical

### Route: bold-proposal

Use for creative proposals, agency pitches, campaign plans, and commercial briefs.

- Backgrounds: cream, yellow, lime, lavender, mint, orange, electric violet, black
- Typography: massive words such as `Brief`, `Approach`, `Scope`, `Timeline`, `Team`, `Invest`
- Layout: one major thought per slide, small tags, thin lines, low horizontal content blocks
- Feel: editorial, confident, colorful, easy to scan

### Route: soft-media

Use for media kits, creator profiles, brand partnerships, portfolios, and lifestyle decks.

- Backgrounds: white, off-white, black
- Accents: pink glow, magenta glow, cyan glow, pastel gradients
- Layout: large name/title, elegant metrics, creator/photo cards, thumbnail strips, package cards
- Feel: refined, personal, stylish, brand-friendly

## Recommended 8-Slide Structure

1. Cover: one strong promise line plus one visual motif.
2. Objective or problem: why this deck matters now.
3. Insight or market: one big number or sharp commercial observation.
4. Big idea or mechanism: the core solution, product, or campaign idea.
5. Channels or workflow: how the idea moves through execution.
6. Timeline or business model: how the plan becomes credible.
7. KPIs, team, or ask: evidence, resources, or decision point.
8. Closing line: a short action-oriented statement.

## Quality Bar

- The slide should have one dominant object.
- The title should feel intentionally designed, not merely enlarged.
- The motif should repeat across the deck.
- Cards should support the message, not become the style.
- Copy should be scannable in a few seconds.
- Dense slides should be balanced by sparse statement slides.
- The final slide should land with a point of view.
